Hugo安装和搭建

备忘 - 以免时间长了忘记一些重要的操作步骤

注：所有站点搭建、部署、配置等相关的命令都在站点目录下执行

Hugo安装

  1. 创建目录“…\Hugo”
  2. 创建分两个分目录“…\Hugo\bin”和“…\Hugo\Sites”
  3. 下载带Hugo（Extended版本）可执行程序的压缩包（Hugo Release）
  4. 将文件解压到“…\Hugo\bin”下
  5. 将“…\Hugo\bin”添加到系统环境变量PATH中
  6. 验证安装，在shell中输入hugo version

站点搭建

创建一个新的站点 

hugo new site 目录名

使用git将主题 下载到站点目录下的themes目录 

git clone github库地址 themes/主题目录名

在本地启动hugo站点 

hugo server -t 主题目录名 --buildDrafts

新建文章（.md格式）（文件夹和文件会储存在站点目录下的content目录中） 

hugo new 文件名或文件路径

站点部署

部署到Github免费仓库下：

创建新的Gtihub远程仓库（注：远程仓库名格式 - 用户名.github.io）（用户名小写）（例：用户名：Dekr0，仓库名：dekr0.github.io）

部署站点（会在站点目录下创建一个public目录）

hugo --theme=主题目录名 --baseUrl="https://远程仓库名/" --buildDrafts

hugo --theme=m10c --baseUrl="https://dekr0.github.io/" --buildDrafts

剩下的是一些常规的git操作：

cd到public目录，使用git进行仓库初始化（会生成.git） 

git init

将public目录的所有内容进行添加 

git add .

将以上操作进行提交 

git commit -m "提交信息"

将本地public目录对Github的远程仓库进行关联

git remote add origin 远程仓库地址.git
git remote add origin https://github.com/Dekr0/dekr0.github.io.git

将所提交的操作推送到Github的远程仓库中 

git push -u origin master
