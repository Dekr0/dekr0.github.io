数据

常量

self explantaory。如果是常量，编译器一般通过用户书写的形式来识别数据类型

C有别于Python的数据类型

long, short, unsigned, signed, void, _Bool, Complex 和 _Imaginary

其中 long, short, unsigned 和 signed 用于提供基本整数类型的变式，如 unsigned short int 和 long long int

字节和字的概念（重申）

字节为常用的计算机存储单位（1 bytes = 8 bit - 标准定义，但C对此有不同的定义）

字为涉及计算机时给定的自然存储单位（如字长位32位，64位）。字长越大，数据转移越快，允许内存访问也更多。

浮点数

浮点数的存储方案，浮点数分成小数和指数部分来表示，且这两部分分开存储。计算机内部使用二进制和2的幂进行储存。

对于一些算术运算，浮点数损失的精度更多。

计算机的浮点数不能表示取键内所有的值，且通常知识实际值的近似值。

int 类型

存储一个 int 类型要占一个机器字长（一般而言）。

printf()函数 和 int 值

字符串中 %d 指明打印整数的值，%d 称为转换说明，指定函数显示一个值所使用的格式

注：大部分函数需要指定参数的数量，但由于 printf() 函数的参数数目不定，编译器不会捕获转换说明的数量于待打印值的数量不相同的错误。

例：

printf("%d %d %d\n", 10) //

// 打印结果：10 16 1650287143. 后两个值为内存中的任意值，内存中的数据不同，编译器管理内存的位置也不同

八进制和十六进制

C 假定常量是十进制数。八进制和十六进制技术系统在于计算机相关的值时方便（不影响数被储存的方式，都以二进制进行编码），因为 8 和 16 是 2 的幂。

如：十进制数 65536，十六进制是 10000

十六进制的每一位数由4位二进制数表示

如：3 -> 0011, 5 -> 0101

在 C 中，使用 0x 或 0X 前缀表示十六进制，0 前缀表示八进制。

在 printf() 中，以八进制表示数字，使用 %o；以十六进制表示数字，使用 %x. 如果要显示各进制的前缀0、0x 和 0X，必须分别使用 %#o、%#x、%#X。

修饰整数类型

附属关键字：short、long 和 unsigned

  • short int ( 等价位 short) 占用存储空间可能比 int 类型少

  • long int ( 等价位 long) 占用存储空间可能比 int 类型多

  • C99: long long int 或 long long, 至少占 64 位

  • unsigned int 或 unsigned 只用于非负值的场合，且范围不同。16 位 unsigned int: 0~65535。正负号的位现在用于表示另一个二进制位

  • C90: unsigned long int / unsigned long 和 unsigned int / unsigned short

  • C99: unsigned long long int / unsigned long long

  • 在任何由符号类型前添加关键字 signed，表示强调使用有符号类型的意图

多种整型需注意的地方

C 语言只规定了 short 占用的储存空间不能多于 int， int 不能多于 long，是为了适应不同的机器。在实际使用中，有些类型之间通常有重叠

如，Windows 3 机器上的 int 和 short 都占 16 位，long 为 32位；后来 Windows 和苹果系统都使用 16位 存储 short，32 位存储 int 和 long。64 位处理器普及后，引入能储存 64 位整数的 long long。

C 标准对基本数据只规定了允许的最小大小。如，16 位机，short 和 int 的最小取值范围都是 [-32767, 32767]

多种 int 类型使用所需注意的地方：

  • 除非在必要的情况下，不要使用 long 类型，因为对于那些 long 类型占用的空间比 int 大的系统，使用 long 类型会减慢运算速度

  • 如果在 long 类型和 int 类型占用空间相同的机器上编码，当确实需要 32 位的整数时， 应使用 long 类型而不是 int 类型。对于 long long，同理

  • 如果在 int 设置为 32 位的系统中使用 16 位的值，使用 short 节省空间

  • 编译器会根据取值范围是否被超过来调整常量的类型（存储空间小至存储空间大）。八进制和十六进制同理

  • 可用 l 或 L 后缀来把常量作为 long 处理; ll 或 LL 后缀来把常量作为 long long 处理; u 或 U 后缀表示 unsigned。这两类后缀的顺序不影响类型分配，且同样能运用到八进制和十六进制中

整数溢出

原理：当变量的值到达到类型能表示最大的时候，会从类型取值范围的最小值重新计数。
