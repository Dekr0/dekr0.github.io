Linux常用指令（一）

    • 进程相关

对于普通用户, 命令行提示符一般为 “$”. 而 root 是以 “#” 结尾

绝对 (Absolute) 路径 - 指定一个相对于系统的根目录 (root directory)的位置. 如果提供的路径以 / 开头, 此路径是以根目录为起始点的绝对路径

相对 (Relatitve) 路径 - 如果路径以 ~/ 开头, 路径起始点为 home 目录; 如果路径以一个文件夹或文件的名字开头，无 / 或 ~/, 此路径是以当前目录为起始点的相对路径; 当前目录可以以 . 或 ./ 表.

pwd

显示当前路径

进程相关

ps

显示当前在系统的进程和进程的信息

kill

向一个或多个进程发送一个信号 (通常来结束一个进程)

参数:

-l: 显示所有信号类型 (后加信号类型名返回其对应的整数值)

-n: 根据信号类型对应的整数值来选择所发送信号的类型

bg

将一个进程放进后台

可以让一个因 Ctrl-Z 被挂起的进程在后台继续运行

fg

将一个进行放进前台

jobs

显示当前的进程 (无详细信息)

&

让命令在后台运行

单条命令 

$ ping www.google.com &

两条或多条命令 

$ ping www.google.com & sudo pacman -Syyu

;

让多条命令按顺序执行（非同时）

$ sudo pacman -Syyu ; cd ~ ; mkdir test

命令返回状态

如果一个命令执行成功, 将返回状态 0; 如果一个命令执行失败, 将返回状态 1

&&

逻辑和运算符, 允许两个或多个指令一起执行 (前提是第一条命令执行成功, 如果是多条命令, 以此类推).

||

逻辑或运算符

$ pacman -Syyu || mkdir test

如果第一条命令执行失败, 则执行第二条命令. 对于多条命令, 则以此类推

!

逻辑否, 类似于 except 语句

命令会执行所有除 ! 后跟的条件

例：移除cmd-test文件夹中所有除 .html 的文件

$ [... cmd-test]$ rm -r !(*.html)

&& … ||

&& 和 || 组合起来相当于一个 if-else 语句

例：

ping 网址 www.google.com, 如果成功, 则控制台显示 ‘Verified’; 如果失败, 则显示 ‘Host Down’.

测试开关网络的结果

$ ping www.google.com && echo "Verified" || echo "Host Down"

连接两条或多条命令, 后跟的命令依赖于前面命令的执行

()

类似于分组

重新定向输入和输出 stream

默认 / 标准 stream (每一个 stream 都有一个与关联的文件描述符, 作用 - 识别):

  • 0 标准输入 (stdin), 程序输入的普通源,
  • 1 标准输出 (stdout), 用于从程序进行普通输出
  • 2 标准错误 (stderr), 用于程序的错误信息和诊断

当标准交互出现在用户和终端时，标准输入会以用户的键盘方式被传入。标准输出和标准错误会以文字的形式被显示到用户的终端中

标准输入 (stdin)

一般地, stdin 会将用户的数据传输到程序中. 等待 stdin 的程序通常会从一个设备, 如键盘, 来接收输入. stdin 会在到达 EOF (end-of-file, 即没有更多的数据被读入) 时候被终止.

例:

使用 cat 打开一个循环的提示符后, 当在提示符中进行输入时, 输入的内容会发送到 stdin 并传输到正在运行的 cat 程序 (等待输入). cat 程序会将输入的内容以 stdout 形式发送给终端

此时, EOF 可以 ctrl-d 的形式输入, 在这之后, cat 收到 EOF 并停止运行

标准输出 (stdout)

stdout 会写程序生成的数据. 当 stdout stream 没有被重新定向, 就会将文字输出到终端中.

标准错误 (stderr)

stderr 会写程序在每一时刻因执行失败的错误. stderr stream 的默认输出目标为终端输出

当一个程序的 stderr 流通过管道传输 (piped) 至第二个程序时, 被管道传输的数据 (包含程序的错误) 将会同时送往终端

流重定向的指令

这些命令会将 stdout 写入一个文件. 如果一个指定文件名不存在, 将会新建文件.

重写

  • > stdout
  • < stdin
  • 2> stderr

追加

  • » stdout
  • « stdin
  • 2» stderr

例:

将 home 目录下的内容重定向至 home.txt 中 

ls ~ | home.txt

command > /dev/null 任何被重定向至 /dev/null 的数据都会被

Pipes

管道是用来将一个 stream 从一个程序重定向到另一个程序. 当一个程序的 stdout 通过管道发送到另外一个程序中, 第一个程序的数据会被另外一个程序接收, 不会被输出到终端中

pipe 命令 

* | *

例:

将当前目录的内容, 管道传输到 less 程序 () 中 

ls | less

pipe 与 > 和 » 的区别是 pipe 从一个命令重定向数据到另一个, 而 > 和 » 专门用来重定向至文件

Filters

Filters 指的是一类能够修改 pipe 的重定向和输出的命令

  • find - 返回符合已传入参数条件的文件及其文件名
  • grep - 返回符合正则表达式的文字
  • tee - 重定向 stdin 至 stdout 以及一个或多个文件
  • tr - 查找一个字符串并替换成另一个字符串
  • wc - 对字符, 行和字进行计算

cat

连接单个或多个文件并将其打印到标准输出中

如果是只有命令本身, cat 会打开一个循环的提示符.

top
